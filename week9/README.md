## 📚 แนวคิดหลักที่เรียนรู้

### ฐานข้อมูล (Database)
ระบบที่ใช้สำหรับจัดเก็บข้อมูลอย่างเป็นระเบียบ เพื่อให้สามารถเรียกใช้งาน แก้ไข และจัดการข้อมูลได้อย่างมีประสิทธิภาพ ในสัปดาห์นี้เราย้ายจากการเก็บข้อมูลในตัวแปร (In-memory) ไปสู่การเก็บข้อมูลถาวร (Persistent Storage)

### SQLite
เป็นฐานข้อมูลแบบ Relational Database (RDBMS) ที่มีลักษณะเด่นคือ "ไม่ต้องมีเซิร์ฟเวอร์" (Serverless) โดยข้อมูลทั้งหมดจะถูกเก็บไว้ในไฟล์เดียว (.db) เหมาะสำหรับการเรียนรู้ การทดสอบ และแอปพลิเคชันขนาดเล็กถึงกลาง

### ภาษา SQL (Structured Query Language)
ภาษามาตรฐานในการจัดการฐานข้อมูล:
- **CREATE TABLE:** สร้างตารางข้อมูล
- **INSERT INTO:** เพิ่มข้อมูลใหม่ลงในตาราง
- **SELECT:** ดึงข้อมูลจากตาราง
- **UPDATE:** แก้ไขข้อมูลที่มีอยู่
- **DELETE:** ลบข้อมูลออกจากตาราง

### การเชื่อมต่อ (Integration)
การใช้ Library เช่น `sqlite3` ใน Node.js เพื่อส่งคำสั่ง SQL จากโค้ด JavaScript ไปยังฐานข้อมูล และรับผลลัพธ์กลับมาประมวลผล

## 🛠️ ขั้นตอนการพัฒนาในสัปดาห์นี้
1.  **Database Configuration:** สร้างไฟล์จัดการฐานข้อมูลและตั้งค่าการเชื่อมต่อ
2.  **Schema Definition:** กำหนดโครงสร้างตาราง `tasks` (id, title, status)
3.  **Refactoring API:** ปรับปรุง API Endpoints จากสัปดาห์ที่ 8 ให้ดึงและบันทึกข้อมูลผ่าน SQLite แทนตัวแปรจำลอง
4.  **Data Persistence Testing:** ทดสอบการปิดเริ่มต้นเซิร์ฟเวอร์ใหม่เพื่อยืนยันว่าข้อมูลยังอยู่ครบถ้วน
