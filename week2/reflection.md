# Lab Reflection - Week 2

## What I Learned
This week was focused on the technical and collaborative aspects of Software Engineering through Git version control.

### Key Takeaways:
1.  **Branching Strategy:** I learned why feature branches are critical for keeping the main codebase stable while developing new ideas.
2.  **Git Commands:** Practiced the core flow of `add`, `commit`, `branch`, and `merge`.
3.  **Collaborative Workflow:** Understood how Pull Requests and Code Reviews work in a team setting. Even in solo development, simulating these steps helps maintain high quality.
4.  **Documentation:** Learned that good commit messages and a clear README are essential for team communication.

### Challenges Faced:
- Simulating a multi-person workflow while working solo requires careful planning of the Git history to show the progression.
- Ensuring the merge process was clean and documented.

## Future Goals
In Week 3, I look forward to moving into Requirements Analysis and User Stories, which will allow me to start applying these Git skills to actual feature development based on user needs.
