# Git Workflow Workshop

## Objective
To understand and practice the feature branch workflow, simulating professional team collaboration.

## Workflow Steps
1.  **Branching:** Create a new branch for the feature.
    - `git checkout -b feature/name`
2.  **Development:** Make changes and commit them.
    - `git add .`
    - `git commit -m "Describe changes"`
3.  **Review:** In a team, this would be the stage for a Pull Request.
4.  **Merging:** Bring the changes back to the main branch.
    - `git checkout main`
    - `git merge feature/name`
