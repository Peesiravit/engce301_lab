# Lab Reflection - Week 5

## สิ่งที่ได้เรียนรู้ (What I Learned)
สัปดาห์นี้ช่วยให้ผมเห็นภาพใหญ่ (Big Picture) ของระบบ ก่อนที่เราจะเริ่มลงมือเขียนโค้ดแต่ละส่วน

### ประเด็นสำคัญ:
1.  **การแยกส่วนความรับผิดชอบ (Separation of Concerns):** ผมเข้าใจแล้วว่าทำไมเราไม่ควรเขียนโค้ดทั้งหมดไว้ในไฟล์เดียว การแยก Layer ช่วยให้เราทำงานได้เป็นระบบและลดความสับสน
2.  **Layered Architecture:** การแบ่งเป็น Presentation, Business, และ Data Layer ช่วยให้ทีมงานที่ดูแล Frontend และ Backend สามารถทำงานขนานกันได้ง่ายขึ้น
3.  **Scalability:** การออกแบบที่ดีในวันนี้ จะช่วยให้เราไม่ต้องรื้อระบบใหม่เมื่อมีผู้ใช้งานเพิ่มขึ้นในวันหน้า
4.  **MVC Pattern:** เข้าใจเหตุผลเบื้องหลังของเฟรมเวิร์กสมัยใหม่ (เช่น React หรือ Express) ว่าทำไมถึงมีโครงสร้างที่คล้ายคลึงกัน

### ความท้าทายที่พบ:
- การตัดสินใจว่าโค้ดส่วนไหนควรอยู่ใน Service หรือ Repository (บางครั้งเส้นแบ่งอาจจะดูจาง)
- การวางผังการไหลของข้อมูล (Data Flow) ให้ไม่ซับซ้อนจนเกินไป
