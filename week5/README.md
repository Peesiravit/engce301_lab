# ปฏิบัติการสัปดาห์ที่ 5: Software Architecture & High Level Design

## 🎯 ภาพรวมรายวิชา
- **วิชา:** ENGCE301 การออกแบบและพัฒนาซอฟต์แวร์ (Software Design and Development)
- **หัวข้อ:** สถาปัตยกรรมซอฟต์แวร์และการออกแบบในระดับสูง

## 📚 แนวคิดหลักที่เรียนรู้

### สถาปัตยกรรมซอฟต์แวร์ (Software Architecture)
โครงสร้างพื้นฐานของระบบซอฟต์แวร์ที่กำหนดองค์ประกอบหลัก ความสัมพันธ์ระหว่างองค์ประกอบ และแนวทางในการทำงานร่วมกัน เปรียบเสมือน "โครงสร้างตึก" ก่อนการตกแต่ง

### สถาปัตยกรรมแบบแยกเลเยอร์ (Layered Architecture)
การแบ่งส่วนประกอบของระบบออกเป็นชั้นๆ ตามความรับผิดชอบ (Separation of Concerns) ทำให้ง่ายต่อการบำรุงรักษาและทดสอบ:
1.  **Presentation Layer (UI):** ส่วนที่โต้ตอบกับผู้ใช้ (เช่น React Component)
2.  **Business Logic Layer (Service):** ส่วนที่ประมวลผลตรรกะของธุรกิจ (เช่น การคำนวณวันเวลางานเสร็จ)
3.  **Data Access Layer (Repository):** ส่วนที่ใช้ติดต่อกับฐานข้อมูล

### รูปแบบการออกแบบ MVC (Model-View-Controller)
- **Model:** จัดการข้อมูลและกฎเกณฑ์ (Data Logic)
- **View:** แสดงผลข้อมูลให้ผู้ใช้เห็น (UI)
- **Controller:** รับคำสั่งจากผู้ใช้และประสานงานระหว่าง Model และ View

## 🛠️ ความสำคัญของการออกแบบสถาปัตยกรรม
- **Scalability:** รองรับการขยายตัวของระบบในอนาคต
- **Maintainability:** ทำให้การแก้ไขโค้ดทำได้ง่ายและลดผลกระทบต่อส่วนอื่นๆ
- **Testability:** แยกส่วนงานเพื่อให้สามารถเขียนหน่วยทดสอบ (Unit Test) ได้ง่ายขึ้น

## 💻 กิจกรรมปฏิบัติการ: การออกแบบสถาปัตยกรรมระบบ
ในสัปดาห์นี้ได้นำ User Stories และ UML จากสัปดาห์ก่อนหน้ามาวางสถาปัตยกรรมในรูปแบบ 3-Tier Layered Architecture สำหรับระบบจัดการงาน
