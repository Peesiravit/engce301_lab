# Lab Reflection - Week 12

## สิ่งที่ได้เรียนรู้ (What I Learned)
สัปดาห์นี้ทำให้ผมเข้าใจถึงความสำคัญของ "ความเร็วและความน่าเชื่อถือ" ในการพัฒนาซอฟต์แวร์ยุคใหม่

### ประเด็นสำคัญ:
1.  **Automation is Key:** จากที่เมื่อก่อนเราต้องรันคำสั่งทดสอบเองทุกครั้งด้วยมือ การมี CI ช่วยให้เรามั่นใจได้ว่าโค้ดที่รวมเข้ามาจะไม่ทำให้ระบบพัง
2.  **GitHub Actions Flexibility:** ไฟล์ YAML ดูเหมือนจะเขียนยากในตอนแรก แต่เมื่อเข้าใจโครงสร้างแล้ว พบว่าเป็นเครื่องมือที่ทรงพลังมาก และสามารถจัดการได้ตั้งแต่การเช็คโค้ดไปจนถึงการ Deploy จริง
3.  **Early Bug Detection:** การรัน CI ในทุกๆ Pull Request ช่วยให้เราตรวจพบข้อผิดพลาดได้ตั้งแต่วันแรกๆ (Shift Left Testing)
4.  **Consistency:** เซิร์ฟเวอร์ CI ช่วยให้เรามั่นใจว่าโค้ดสามารถรันได้ในสภาพแวดล้อมมาตรฐาน (Standard Environment) ไม่ใช่แค่รันได้บนเครื่องนักพัฒนาคนใดคนหนึ่ง (Works on my machine)

### ความท้าทายที่พบ:
- การตั้งค่าเวอร์ชันของ Node.js และการจัดการ Cache เพื่อให้ CI ทำงานได้รวดเร็วขึ้น
- การตีความผลลัพธ์จาก Pipeline (Log messages) เมื่อเกิดข้อผิดพลาดในขั้นตอนต่างๆ

## เป้าหมายในอนาคต
ในสัปดาห์ถัดๆ ไป ผมหวังว่าจะได้ลองเชื่อมต่อส่วน CD (Continuous Deployment) เข้ากับคลาวด์แพลตฟอร์มจริง เช่น Railway หรือ Vercel เพื่อให้ซอฟต์แวร์ของเราออนไลน์ให้คนอื่นใช้งานได้จริงๆ ครับ
