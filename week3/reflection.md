# Lab Reflection - Week 3

## สิ่งที่ได้เรียนรู้ (What I Learned)
สัปดาห์นี้เน้นไปที่จุดเริ่มต้นที่สำคัญที่สุดของการพัฒนาซอฟต์แวร์ นั่นคือการรู้ว่าเรากำลังสร้างอะไรเพื่อใคร

### ประเด็นสำคัญ:
1.  **ความต้องการที่ชัดเจน:** ผมได้เรียนรู้ว่าความยากไม่ได้อยู่ที่การเขียนโค้ด แต่อยู่ที่การตีความความต้องการของผู้ใช้ให้กลายเป็นคุณสมบัติที่จับต้องได้
2.  **พลังของ User Stories:** การใช้รูปแบบ "As a... I want... So that..." ช่วยให้เราโฟกัสที่ "คุณค่า" ที่ผู้ใช้จะได้รับ มากกว่าแค่ฟังก์ชันการทำงานเฉยๆ
3.  **เกณฑ์การยอมรับ (Acceptance Criteria):** สิ่งนี้ช่วยลดความคลุมเครือระหว่างผู้พัฒนาและผู้สั่งงาน (Stakeholders) ได้อย่างมาก
4.  **การจัดลำดับความสำคัญ:** เข้าใจว่าทำไมเราถึงต้องมี Backlog และการเลือกทำสิ่งที่สำคัญที่สุดก่อน

### ความท้าทายที่พบ:
- การเขียน User Story ให้สั้น กระชับ แต่อยู่ในขอบเขตที่เหมาะสม (Granularity)
- การกำหนดเกณฑ์การยอมรับที่สามารถนำไปทดสอบได้จริง (Testable)

## เป้าหมายในอนาคต
ในสัปดาห์ที่ 4 ผมจะนำ User Stories เหล่านี้ไปพัฒนาต่อเป็นการจำลองโมเดลระบบ (System Modeling) และการออกแบบ UML เพื่อให้เห็นโครงสร้างทางเทคนิคที่ชัดเจนขึ้น
