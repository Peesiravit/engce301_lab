# รายการ User Stories: ระบบจัดการงาน (Task Management System)

## 👤 ตัวละครหลัก (User Roles)
1.  **ผู้ใช้งานทั่วไป (Member):** ผู้ที่ต้องการจัดการงานของตนเอง
2.  **หัวหน้าทีม (Team Lead):** ผู้ที่ต้องการภาพรวมของงานในทีม

---

## 📋 รายการ User Stories

### 1. การสร้างรายการงานใหม่
**User Story:** ในฐานะ *ผู้ใช้งานทั่วไป* ฉันต้องการ *สร้างรายการงานใหม่ได้* เพื่อที่ฉันจะได้ *บันทึกสิ่งที่ต้องทำไว้ในระบบ*

**Acceptance Criteria (เกณฑ์การยอมรับ):**
- [ ] มีปุ่ม "สร้างงานใหม่" ที่เห็นได้ชัดเจน
- [ ] มีช่องให้กรอกชื่อหัวข้องาน (บังคับกรอก)
- [ ] มีช่องให้กรอกรายละเอียด (ไม่บังคับ)
- [ ] เมื่อบันทึกสำเร็จ งานจะปรากฏในรายการ "งานที่ต้องทำ" ทันที

### 2. การเปลี่ยนสถานะงาน
**User Story:** ในฐานะ *ผู้ใช้งานทั่วไป* ฉันต้องการ *เปลี่ยนสถานะงานจาก "กำลังทำ" เป็น "เสร็จสิ้น"* เพื่อที่ฉันจะได้ *ติดตามความคืบหน้าของตนเอง*

**Acceptance Criteria (เกณฑ์การยอมรับ):**
- [ ] สามารถลากและวาง (Drag & Drop) งานระหว่างคอลัมน์ได้
- [ ] มีตัวเลือกให้คลิกเปลี่ยนสถานะในหน้ารายละเอียดงาน
- [ ] ระบบบันทึกเวลาที่งานเสร็จสมบูรณ์โดยอัตโนมัติ

### 3. การมอบหมายงาน
**User Story:** ในฐานะ *หัวหน้าทีม* ฉันต้องการ *มอบหมายงานให้สมาชิกในทีมได้* เพื่อที่ฉันจะได้ *กระจายภาระงานได้อย่างทั่วถึง*

**Acceptance Criteria (เกณฑ์การยอมรับ):**
- [ ] มีรายชื่อสมาชิกโครงการให้เลือกในตอนสร้างหรือแก้ไขงาน
- [ ] สมาชิกที่ได้รับมอบหมายงานจะได้รับข้อความแจ้งเตือน (Notification)
- [ ] งานที่มอบหมายแล้วจะแสดงรูปโปรไฟล์ของผู้รับผิดชอบบนการ์ดงาน
