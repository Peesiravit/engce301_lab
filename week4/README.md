## 📚 แนวคิดหลักที่เรียนรู้

### การจำลองโมเดลระบบ (System Modeling)
กระบวนการสร้างตัวแทนของระบบ (Model) เพื่อช่วยให้ผู้พัฒนาและผู้ที่เกี่ยวข้องเข้าใจโครงสร้างและพฤติกรรมของซอฟต์แวร์ก่อนเริ่มเขียนโค้ดจริง

### Unified Modeling Language (UML)
ภาษามาตรฐานสำหรับการสร้างภาพอธิบาย (Visualizing) การระบุรายละเอียด (Specifying) และการจัดทำเอกสารสำหรับระบบซอฟต์แวร์

### แผนภาพ UML ที่สำคัญในสัปดาห์นี้
1.  **Use Case Diagram:** แสดงการโต้ตอบระหว่างผู้ใช้ (Actors) และระบบ เพื่อให้เห็นขอบเขตรวมของฟังก์ชันการใช้งาน
2.  **Class Diagram:** แสดงโครงสร้างคงที่ของระบบ (Static Structure) ประกอบด้วย Class, Attributes, Methods และความสัมพันธ์ระหว่างกัน
3.  **Sequence Diagram:** แสดงการโต้ตอบระหว่างวัตถุ (Objects) ตามลำดับเวลา เพื่อให้เห็นขั้นตอนการทำงานของระบบในฟังก์ชันหนึ่งๆ

## 🛠️ เครื่องมือที่ใช้
