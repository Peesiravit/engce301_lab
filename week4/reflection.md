# Lab Reflection - Week 4

## สิ่งที่ได้เรียนรู้ (What I Learned)
สัปดาห์นี้เป็นการเปลี่ยนจากนามธรรม (Requirements) ให้กลายเป็นรูปธรรม (Visual Models) ผ่าน UML

### ประเด็นสำคัญ:
1.  **พลังของการสร้างภาพ (Visualization):** การวาดแผนภาพช่วยให้เราเห็นจุดบกพร่องในตรรกะของระบบที่เราอาจมองข้ามไปเมื่อเขียนเป็นข้อความ
2.  **Use Case Diagram:** ช่วยให้เราเห็นขอบเขต (Boundary) ของระบบที่ชัดเจนว่าใครทำอะไรได้บ้าง
3.  **Class Diagram:** เป็นพิมพ์เขียวที่สำคัญมากก่อนการเขียนโค้ด เพราะช่วยให้เราออกแบบโครงสร้างความสัมพันธ์ของข้อมูล (Data Relationships) ได้อย่างถูกต้อง
4.  **Sequence Diagram:** เมื่อเราต้องออกแบบระบบที่มีการโต้ตอบที่ซับซ้อน แผนภาพนี้ช่วยให้เราลำดับขั้นตอนและจัดการการแจ้งเตือน (Notifications) ได้อย่างเป็นระบบ

### ความท้าทายที่พบ:
- การเลือกความสัมพันธ์ที่ถูกต้องใน Class Diagram (เช่น เมื่อไหร่ควรใช้ Association, Aggregation หรือ Composition)
- การเขียน Mermaid syntax ให้ถูกต้องเพื่อจัดการแผนภาพที่ซับซ้อน
