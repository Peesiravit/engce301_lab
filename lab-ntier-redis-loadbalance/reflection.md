# Lab Reflection - Week 13

## สิ่งที่ได้เรียนรู้ (What I Learned)
สัปดาห์นี้เป็นการเปิดหูเปิดตาเรื่องความสามารถในการขยายระบบ (Scalability) และการเตรียมพร้อมรับมือกับเหตุการณ์ไม่คาดฝัน (Availability)

### ประเด็นสำคัญ:
1.  **Orchestration in Action:** การเห็น `docker-compose` สั่งรัน Replica ได้หลายตัว ทำให้ผมเข้าใจว่าระบบใหญ่ๆ จัดการเรื่องการรับโหลดมหาศาลได้อย่างไร
2.  **The Role of Nginx:** Nginx ไม่ได้เป็นแค่เว็บเซิร์ฟเวอร์ แต่เป็น "พนักงานต้อนรับ" ที่จัดคิวและคอยเช็คว่าพนักงานคนไหน (App Instance) พร้อมให้บริการบ้าง 
3.  **Failover Importance:** การตั้งค่า `proxy_next_upstream` ช่วยลดความเสี่ยงที่ผู้ใช้จะเจอหน้าขาวหรือ 502 Bad Gateway เพราะระบบจะดีด Request ไปหาตัวที่ยังรอดชีวิตอยู่โดยอัตโนมัติ
4.  **Redis vs Database:** เข้าใจว่าทำไมต้องมี Cache เพราะการดึงข้อมูลจากแรมไวว่าการดึงจากดิสก์ (Database) มาก ซึ่งช่วยลด Latency ได้อย่างมหาศาลเมื่อระบบมีโหลดเยอะ

### ความท้าทายที่พบ:
- การตั้งค่า Network ใน Docker ให้ Nginx มองเห็นชื่อ Service ของ App ได้ทุุกตัว
- การจำลองสถานการณ์ Server ล่ม และต้องคอยสังเกต Log ของ Nginx ว่ามีการส่งต่อ Request ไปยังตัวอื่นจริงหรือไม่
