# ปฏิบัติการสัปดาห์ที่ 6: Low Level Design ระดับพื้นฐาน โดยใช้ React.js

## 🎯 ภาพรวมรายวิชา
- **วิชา:** ENGCE301 การออกแบบและพัฒนาซอฟต์แวร์ (Software Design and Development)
- **หัวข้อ:** การออกแบบระดับรายละเอียด (Low Level Design) และพื้นฐาน React.js

## 📚 แนวคิดหลักที่เรียนรู้

### การออกแบบระดับรายละเอียด (Low Level Design - LLD)
กระบวนการขยายความจากสถาปัตยกรรม (High Level) ไปสู่รายละเอียดของแต่ละส่วนงาน เช่น การกำหนดโครงสร้างของคลาส (Classes), เมธอด (Methods) หรือในบริบทของ React คือการกำหนดลำดับขั้นของส่วนประกอบ (Component Hierarchy)

### พื้นฐาน React.js สำหรับ LLD
1.  **Components:** ส่วนประกอบย่อยของ UI ที่ทำงานเป็นอิสระต่อกัน (เช่น Button, Card, Navbar)
2.  **Props (Properties):** ข้อมูลที่ส่งจากส่วนประกอบแม่ไปยังส่วนประกอบลูก (Read-only)
3.  **State:** ข้อมูลภายในส่วนประกอบที่สามารถเปลี่ยนแปลงได้ และทำให้เกิดการวาดหน้าจอใหม่ (Re-render)
4.  **Component Hierarchy:** การวางผังว่าส่วนประกอบไหนเป็นลูกของส่วนประกอบไหน

### แรงบันดาลใจในการออกแบบ (Modern UI)
เพื่อให้ตรงตามมาตรฐาน "Premium Look" เราจะใช้เทคนิค:
- **Functional Components:** การเขียนส่วนประกอบด้วยฟังก์ชัน
- **Atomic Design:** การแยกส่วนประกอบให้เล็กที่สุดเพื่อให้นำกลับมาใช้ใหม่ได้ง่าย
