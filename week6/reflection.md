# Lab Reflection - Week 6

## สิ่งที่ได้เรียนรู้ (What I Learned)
สัปดาห์นี้เป็นการนำทฤษฎีสถาปัตยกรรมมาเปลี่ยนเป็นโค้ดที่ทำงานได้จริงผ่าน React.js

### ประเด็นสำคัญ:
1.  **การมอง UI เป็นส่วนประกอบ (Thinking in React):** ผมได้เรียนรู้วิธีการย่อยหน้าจอปลาทู (App) ให้กลายเป็นชิ้นเล็กๆ (Components) ซึ่งช่วยให้โค้ดสะอาดและแก้ไขง่ายขึ้นมาก
2.  **ความสำคัญของ LLD:** การวาดผัง Component Hierarchy ก่อนเขียนโค้ดช่วยลดความสับสนเรื่องการส่งข้อมูล (Props) ได้อย่างชัดเจน
3.  **การจัดการข้อมูล:** เข้าใจความแตกต่างระหว่าง Props (ข้อมูลที่ได้รับมา) และ State (ข้อมูลที่ตัวเราจัดการเอง) ซึ่งเป็นหัวใจหลักของ React
4.  **Premium UI ใน React:** การประยุกต์ใช้ Inline Styling หรือ CSS-in-JS เพื่อทำ Glassmorphism ใน Component ช่วยให้งานดูเป็นมืออาชีพมากขึ้น

### ความท้าทายที่พบ:
- การตัดสินใจเลือกจุดที่จะเก็บ "State" (Lifting State Up) เพื่อให้ส่วนประกอบต่างๆ ใช้งานข้อมูลร่วมกันได้อย่างถูกต้อง
- การออกแบบ Component ให้มีความยืดหยุ่น (Reusable) เพื่อให้นำไปใช้ซ้ำในจุดอื่นๆ ได้โดยไม่ต้องเขียนใหม่
